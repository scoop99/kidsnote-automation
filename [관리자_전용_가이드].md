# 관리자 전용 가이드 (Secret)

이 파일은 지인들에게 배포할 때 포함되지 않으며, GitHub에도 올라가지 않도록 설정되었습니다. 오직 관리자님만 확인하세요.

## 📦 배포 (Distribution) 방법
지인들에게 처음 프로그램을 줄 때의 과정입니다.

1. **폴더 정리**:
   - `system/config.json`을 열어 `github_repo`가 `scoop99/kidsnote-automation`으로 되어 있는지 확인합니다.
   - `system/session.json` 파일이 있다면 삭제해 주세요. (지인이 자신의 아이디로 로그인해야 하므로 내 세션은 지워야 합니다.)
2. **압축**: 
   - 프로젝트 전체 폴더를 ZIP으로 압축합니다.
   - **팁 (중요)**: `system/node_modules` 폴더를 포함해서 압축하면, 지인이 처음 실행할 때 라이브러리 설치 과정을 건너뛸 수 있어 매우 빨라집니다. (용량은 커지지만 훨씬 편리함)
   - `PLAN-*.md`, `.git` 폴더, `.gitignore` 등은 굳이 포함하지 않아도 작동에 지장이 없습니다.
3. **전달**: 압축 파일을 카톡이나 구글 드라이브 등으로 지인에게 전달합니다.

## 🔄 코드 업데이트 방법 (자동 업데이트 로직)
키즈노트 사이트가 변해서 코드를 수정했을 때의 과정입니다. 지인들에게 파일을 다시 줄 필요가 없습니다!

1. **코드 수정**: 필요한 로직을 수정합니다.
2. **버전 올리기**: `system/package.json` 파일의 `"version": "1.0.0"`을 `"1.0.1"` 등으로 올립니다.
3. **GitHub Push**: `git add .`, `git commit`, `git push`를 통해 코드를 GitHub에 올립니다.
4. **Release 생성 (매우 중요)**:
   - GitHub 사이트에서 `Create a new release`를 클릭합니다.
   - 태그 이름을 `v1.0.1` (버전과 일치)로 적고 배포합니다.
5. **완료**: 지인들이 프로그램을 평소처럼 실행하면, 코드가 자동으로 최신 버전으로 교체됩니다.

## 🛠️ 유지보수 팁
- `updater.js`는 `system/` 폴더 내의 파일들을 통째로 교체합니다. 
- 배포 시 `.gitignore`에 있는 파일(`config.json` 등)은 업데이트 시에도 덮어쓰지 않고 보존되도록 구현되어 있습니다.

## 🛡️ 안정성 및 인코딩 (Critical)
- **런처 구조**: `[여기를_더블클릭하세요].bat`은 오직 영문으로만 구성된 파워쉘 호출기입니다. 실제 로직은 `system/scripts/launcher.ps1`에서 처리됩니다. (인코딩 문제 원천 차단)
- **표준 인코딩**: 모든 스크립트(`.js`, `.ps1`)는 반드시 **UTF-8 with BOM** 형식으로 저장해야 합니다. 그렇지 않으면 한국어 윈도우 터미널에서 구문 오류나 글자 깨짐이 발생할 수 있습니다.
